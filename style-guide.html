01

Logo

Use the full-color PWI logo on a white field. Keep it clear, flat, and easy to read.

Approved Use

Rules

  • Place the mark on white or in a white logo block when the surrounding area is dark.
  • Keep enough clear space around the logo so it does not touch other UI or page edges.
  • Do not stretch, recolor, add effects, or place the logo on busy images.
  • Use the site header size as the default: roughly 44px tall inside the white block.

Forms & Inputs

The pattern already used on contact-form.html. Reuse these exact classes on every new form instead of inventing new input styles.

Default & Focus

  • Always pair a real <label for=""> with its input id — never rely on placeholder text alone.
  • Mark required fields with a red asterisk in the label, not in the placeholder.
  • Focus ring is focus:ring-2 focus:ring-[#1E4ED8]/30 on every field — do not remove outlines without replacing them.
  • Inputs are rounded-md with a 1px border-line border — match the radius used on cards (8px), not the 9999px pill radius used on buttons.

Disabled & Error (not yet used on the site — adopt this pattern the next time a form needs them)

Enter a valid email address.

  • Disabled: background:#f1f5f9, muted text color, cursor:not-allowed, no focus ring (the field can't be focused).
  • Error: 1px solid #dc2626 border instead of border-line, plus a short red helper line below the field, linked with aria-describedby and aria-invalid="true" on the input.
  • Never use color alone to signal an error — the helper text is what makes it accessible to colorblind users and screen readers.

07

Imagery

Product and aircraft imagery should be clear, specific, and inspection-friendly. Avoid generic mood images when the user needs details.

Image Rules

  • Show the actual product, aircraft area, installation context, or approval document.
  • Keep product photos bright enough to inspect shape, finish, and fit.
  • Use dark overlays only when text must sit over a photo.
  • Crop consistently and avoid cutting off important product details.
08

Responsive Scale

Hero spacing and large-display scaling should use viewport-aware values so pages do not feel cramped on mobile or undersized on Studio Display-class screens.

Product Hero Gap Keep product/category hero text clear of the fixed header on every breakpoint.
padding-top: clamp(6.5rem, 6vw, 7.5rem)
mobile: 6.5rem
Homepage Hero Base homepage hero should breathe without pushing all discovery content away.
--home-hero-height: clamp(430px, 42vh, 520px)
Large Screens At 1600px+, scale the hero, lede, CTAs, search, capability cards, header spacing, and footer width together.
--home-hero-height: clamp(520px, 43vh, 640px)
Header & Footer Do not leave global chrome capped to laptop widths on large displays.
.pwi-footer-grid: max-width min(92rem, 100% - 8rem)
Capability Strip Image cards should grow with the viewport and keep readable overlay text.
--home-card-strip-height: clamp(390px, ..., 520px)
09

Markup Hygiene

Keep pages valid for Pinegrow and browsers. These rules reflect the June 2026 cleanup pass across the root HTML files.

Full Pages Every standalone HTML page needs one document wrapper.
<html>, <head>, and <body> each open and close once.
Structural Tags Keep header, main, section, footer, and page scripts inside body.
No Duplicate Attributes Merge duplicate class or style attributes into one attribute so validators do not drop values.
Tables Keep table rows inside valid thead, tbody, or tfoot wrappers. Avoid broken class strings on table containers.
10

Usage Checklist

A short final pass before publishing or building a new page.

Use the full-color logo on a white field.
Keep primary CTAs blue and limit them to one main action per section.
Use dark hero sections for product pages and light heroes for utility pages.
Match desktop and mobile hero top gaps; product hero text should not tuck under the fixed header.
Scale homepage hero, capability cards, header spacing, and footer width at large display widths.
Validate source markup before handoff: no extra closing tags, broken table wrappers, or duplicate attributes.
Keep cards simple: 8px radius, subtle border, no nested card layouts.
Use real product, aircraft, or document images wherever possible.
Keep text concise, technical, and easy to scan.
